# key-value存储

https://developer.aliyun.com/article/376606

高速缓存数据库
Remote Dictionary Server

key-value存储系统
存储形式组成一个键值对，key可以对应 rowid或者定义key 形如 `user:user_1`，可以写成json格式。
这种数据库的优点
是性能高，读写快，
轻松横向扩展

缺点是
查询能力有限
缺乏原子性



## redis

优点：
1. 存储在内存中，查询速度非常快（没有机械读写操作）
2. 支持5种数据类型 （string，list,hash,set,Zset）
3. 优化了查询引擎，多线程查询
4. 原子性，支持事务

5. 高可用，支持主从复制(实现读写分离)和Sentinel(监控主节点)机制
6. 持久化，快照(每个一段时间将数据集保存到磁盘) 和日志追加（记录所有的操作命令）

查询快，应为是精确匹配，扫描key得出对应的值，复杂度是O(1)

缺点：
1. 受内存影响，容易导致系统崩溃

主要用途：
缓存，会话存储，消息队列，排行榜等 需要高频次读写的场景

解决了
sql的在海量数据下，读写慢的问题 ，对磁盘IO的开销小 
redis缓存热门数据，降低了sql的高并发压力
