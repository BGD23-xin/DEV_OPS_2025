# key-value存储

https://developer.aliyun.com/article/376606

高速缓存数据库
Remote Dictionary Server

key-value存储系统
存储形式组成一个键值对，key可以对应 rowid或者定义key 形如 `user:user_1`，可以写成json格式。
这种数据库的优点
是性能高，读写快，
轻松横向扩展

缺点是
查询能力有限（简单查询，对于如SQL的JOIN、复杂的聚合就比较弱）
缺乏原子性(回滚，程序逻辑错误引发的)



## redis

开源内存数据库，高性能的键值存储系统，常用于缓存、消息队列、会话存储等应用场景

优点：
1. 读写都在内存中，查询速度非常快(支持每秒数十万次的读写)
2. 高并发能力 采用了非阻塞I/O多路复用机制（I/O多路复用机制：I/O多路复用就是只有单个线程，通过跟踪每个I/O流的状态，来管理多个I/O流）

查询快，应为是精确匹配，扫描key得出对应的值，复杂度是O(1)

缺点：
1. 其容量受到物理内存的限制
2. 综合能力不如mysql (数据结构，复杂查询的能力)

<!-- 2. 单线程模式(网络io，数据读写) -->

主要用途：
缓存，会话存储，消息队列，排行榜等 需要高频次读写的场景

解决了
sql的在海量数据下，读写慢的问题 ，对磁盘IO的开销小 
redis缓存热门数据，降低了sql的高并发压力
扩展性差
