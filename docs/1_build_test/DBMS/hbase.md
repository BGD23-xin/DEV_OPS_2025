# 列式数据库

https://zhuanlan.zhihu.com/p/662986341

https://www.cnblogs.com/wxdlut/p/17277207.html

故名思义就是以列来存储数据，没有固定的表结构
列式数据库的优点
1. 高压缩率(一列数据类型一致，可压缩)
2. 高效的查询和聚合(因为一列是天然的索引，查询速度会很快)
3. 适合OLAP(分析型业务)
和
缺点
1.写入性能低（一行数据要拆成多个部分写入）
2.不适合事务型操作
3.对硬件资源要求高


## Hbase vs cassandra

在`CAP`的三个原理中
cassandra的一致性较弱，去中心化后，多节点互相复制，最终一致性（各个节点在写入时数据可以不一致），但高可用和分区容错

hbase具有一致性（将数据写入后会写入hdfs，并使节点一致）和分区容错性（主节点会将数据复制到所有节点），高可用低（依赖主节点）

`性能方面`区别
- 延迟：持续写入时，cassandra的延迟会上升(因为写入多个副本节点，还需等待副本节点确认)，hbase的延迟会下降，因为写入一个节点，然后hdfs会复制到其他节点
- 吞吐，cassandra平均单节点的吞吐高于hbase
- 安全性 hbase强(单元格级加密)，Cassandra（调节对记录行级别的访问权限）
- 分区： cassandra支持分区，hbase不支持分区


使用场景：
Cassandra 适用于需要频繁写入数据的应用场景，但它没有针对频繁更新或删除数据进行优化

HBase 更适合需要数据一致性和频繁处理的应用程序

解决了 sql的 大规模、高并发、分布式场景
